# 烹饪助手 产品需求文档（PRD）

## 1. 功能概述

### 1.1 功能定位
家庭日常烹饪助手，解决"今天吃什么"和"这道菜怎么做"两大核心痛点。支持 AI 智能推荐菜品、菜谱浏览收藏、分步做菜引导、食材一键加入购物清单等能力。

### 1.2 目标用户
- 每天需要为家人做饭、不知道吃什么的家庭成员
- 厨艺新手，需要详细做菜步骤指导
- 希望根据现有食材快速找到菜谱的用户
- 想要记录和分享家庭私房菜的用户

### 1.3 核心价值
- **解决选择困难**：AI 根据口味偏好、食材、人数智能推荐菜品
- **手把手教做菜**：分步骤大字展示，做菜模式解放双手
- **打通购物清单**：缺少的食材一键加入购物清单，形成闭环
- **家庭菜谱共享**：私房菜家庭成员共享，传承家的味道

---

## 2. 信息架构

### 2.1 页面层级

```
工具箱 → 烹饪助手
├── 菜谱首页 (cooking/index)         ← 推荐 + 分类浏览 + 搜索
├── 菜谱详情 (cooking/detail)        ← 食材清单 + 做法步骤
├── 做菜模式 (cooking/stepmode)      ← 分步大字展示，解放双手
├── 我的收藏 (cooking/favorites)     ← 收藏的菜谱
├── 家庭菜谱 (cooking/family-recipes) ← 家庭自创菜谱
└── AI 推荐 (cooking/recommend)      ← 智能推荐菜品
```

### 2.2 入口
- 工具箱 → 生活工具 → 烹饪助手（主入口）
- AI 聊天中询问做菜相关问题，引导跳转
- 购物清单页 → "用这些食材做什么菜"（反向联动）

#### 3.2.2 功能说明

| 功能 | 说明 |
|------|------|
| **口味选择** | 可选标签，支持多选 |
| **食材输入** | 自由输入已有食材，AI 会优先推荐用这些食材能做的菜 |
| **快捷食材** | 常见食材标签，点击快速添加 |
| **人数选择** | 影响推荐的分量和菜品数量 |
| **时间筛选** | 筛选烹饪时长，方便快速出餐 |
| **推荐结果** | AI 返回 3-5 个菜品推荐，含简要信息 |
| **换一批** | 不满意可重新生成推荐 |
| **查看菜谱** | 跳转菜谱详情页 |

#### 3.2.3 AI 推荐逻辑

```
用户选择 口味:辣 + 食材:鸡蛋,土豆 + 人数:2人 + 时间:≤30min
  → 调 POST /api/cooking/recommend
  → AI 综合条件推荐菜品
  → 返回菜品列表（含名称、时长、难度、简介）
  → 如果菜谱库有匹配，直接关联；如果没有，AI 生成菜谱
```

## 4. 与现有模块联动

### 4.1 购物清单联动

```
菜谱详情 → 点击"缺料加入购物清单"
  → 弹出食材勾选列表
  → 用户勾选缺少的食材
  → 调用购物清单的 POST /api/shopping/items 批量添加
  → 提示"已添加 N 项到购物清单"
```

### 4.2 AI 聊天联动

```
AI 聊天中用户问"鸡蛋能做什么菜"
  → AI 回答推荐菜品
  → 回答底部附带 [查看菜谱] 快捷按钮
  → 点击跳转烹饪助手对应菜谱
```

### 4.3 购物清单反向联动

```
购物清单页 → "用这些食材做什么"
  → 读取当前清单中的食材类商品
  → 跳转 AI 推荐页，自动填入食材
  → 推荐可做的菜品
```

## 5. 数据模型

### 5.1 数据库表设计

```
Recipe（菜谱）
├── id              主键
├── name            菜名
├── cover_image     封面图 URL
├── description     简介
├── cook_time       烹饪时长（分钟）
├── servings        几人份
├── difficulty      难度：easy / medium / hard
├── category        分类：home / quick / soup / breakfast / cold / baking / baby / diet
├── tags            标签（JSON 数组，如 ["辣","下饭"]）
├── tips            小贴士（文本）
├── source          来源：system(系统预置) / family(家庭创建) / ai(AI生成)
├── family_id       所属家庭（family 来源时）
├── created_by      创建人（user_id）
├── is_public       是否公开（系统菜谱为 true）
├── favorite_count  收藏数
├── created_at      创建时间
└── updated_at      更新时间

RecipeIngredient（菜谱食材）
├── id              主键
├── recipe_id       所属菜谱
├── name            食材名称
├── amount          用量（如 "3个"、"适量"）
├── group_name      分组（主料 / 调料 / 辅料）
└── sort_order      排序

RecipeStep（菜谱步骤）
├── id              主键
├── recipe_id       所属菜谱
├── step_number     步骤序号
├── content         步骤内容
├── tip             本步骤小贴士（可选）
└── image_url       步骤图片（可选）

RecipeFavorite（菜谱收藏）
├── id              主键
├── user_id         用户 ID
├── recipe_id       菜谱 ID
└── created_at      收藏时间
```

### 5.2 关键关系
- Recipe 一对多 RecipeIngredient（一个菜谱有多种食材）
- Recipe 一对多 RecipeStep（一个菜谱有多个步骤）
- User 多对多 Recipe（通过 RecipeFavorite 收藏关系）
- 家庭菜谱通过 family_id 关联，同家庭成员共享

## 7. 交互细节

### 7.1 搜索流程

```
用户输入 "土豆"
  → 前端防抖 300ms 后调 GET /api/cooking/recipes?keyword=土豆
  → 返回菜名或食材包含"土豆"的菜谱列表
  → 展示搜索结果
```

### 7.2 AI 推荐流程

```
用户选择条件 → 点击"开始推荐"
  → 调 POST /api/cooking/recommend
  → AI 返回 3-5 个推荐菜品
  → 如果菜谱库有匹配的 recipe_id，点击直接跳转详情
  → 如果是 AI 新生成的菜谱，自动入库后跳转详情
  → 不满意可点"换一批"重新推荐
```

### 7.3 做菜模式流程

```
菜谱详情 → 点击"开始做菜"
  → 进入全屏做菜模式
  → 自动开启屏幕常亮（wx.setKeepScreenOn）
  → 从 Step 1 开始，大字居中展示
  → 左右滑动或点击按钮切换步骤
  → 最后一步点击"完成"→ 弹出"做好啦 🎉"
  → 退出做菜模式，关闭屏幕常亮
```

### 7.4 缺料加购物清单流程

```
菜谱详情 → 点击"缺料加入购物清单"
  → 弹出食材列表，默认全选
  → 用户取消勾选家里已有的食材
  → 点击确认
  → 调 POST /api/cooking/add-to-shopping
  → 后端调用购物清单模块批量添加
  → 提示"已添加 5 项到购物清单"
```

### 7.5 AI 生成家庭菜谱流程

```
添加家庭菜谱页 → 点击"AI 帮你生成菜谱"
  → 弹出输入框，用户描述做法
  → 调 POST /api/cooking/ai-generate
  → AI 返回结构化菜谱（食材 + 步骤 + 小贴士）
  → 自动填充到表单中
  → 用户可修改后保存
```

## 11. 技术要点

### 11.1 前端
- 做菜模式使用 `wx.setKeepScreenOn({keepScreenOn: true})` 保持屏幕常亮
- 步骤切换支持左右滑动手势（bindtouchstart / bindtouchend）
- 菜谱列表使用分页加载，避免一次性加载过多数据
- 搜索使用防抖（300ms），减少接口请求

### 11.2 后端
- AI 推荐和 AI 生成菜谱复用现有 DashScope API 能力
- 菜谱搜索支持菜名和食材名的模糊匹配
- 预置菜谱通过 seed 脚本批量导入
- 食材加入购物清单复用 shopping 模块的接口逻辑

### 11.3 数据
- 系统预置菜谱 source="system"，不可被用户编辑/删除
- 家庭菜谱 source="family"，仅创建人和同家庭成员可见
- AI 生成菜谱 source="ai"，生成后自动入库
