# PRD：拍照识别植物功能升级 + PC 端适配

> 版本：v1.1 | 日期：2026-03-11

---

## 1. 背景与现状

Plant Sprite 已具备基础的拍照识别功能：
- 用户点击「📷 拍照识别植物」按钮，选择图片后调用通义千问 VL 模型
- AI 返回 `name`（名称）、`watering_interval`（浇水天数）、`category`（室内/室外）、`note`（一句话建议）
- 自动填入表单，用户确认后保存

**当前问题：**
- AI 返回的 `note` 仅是一句简短建议，信息量不足
- 缺少植物的基本描述（外观特征、品种介绍等）
- 缺少结构化的养护方案（光照、温度、施肥、土壤等维度）
- 用户拿到的信息不够丰富，无法指导日常养护
- 页面为纯移动端布局，PC 浏览器打开时内容拉伸至全屏，阅读体验差

## 2. 需求目标

**需求一：AI 识别升级**
- 自动识别植物种类，填入名称、浇水周期、分类
- 生成 **植物描述**（品种介绍、外观特征）和 **详细养护建议**（光照、温度、湿度、施肥、土壤、注意事项）
- 将描述和建议组合后写入 **备注** 字段，方便用户随时查看

**需求二：PC 端适配**
- 页面在 PC 浏览器上有良好的布局和阅读体验
- 内容居中、合理限宽，植物卡片多列排列，充分利用大屏空间

## 3. 用户故事

> **故事 1（AI 识别）：** 作为一个植物爱好者，我拍了一张家里植物的照片，希望 APP 不仅能告诉我这是什么植物、多久浇一次水，还能告诉我这种植物长什么样、怎么养护（需要多少光照、适合什么温度、怎么施肥），这些信息自动保存在备注里，以后随时可以翻看。

> **故事 2（PC 适配）：** 作为一个在电脑上管理植物的用户，我希望在 PC 浏览器打开时页面不会拉得很宽很空，植物卡片能多列排列，弹窗居中显示，整体看起来像一个正常的桌面应用。

## 4. 功能设计

### 4.1 自动填充的表单字段

拍照识别后，以下字段全部自动填入表单，用户可手动修改后保存：

| 表单字段 | 自动填充内容 | 说明 |
|----------|-------------|------|
| **植物名称** | AI 识别的中文名称 | 如"绿萝"、"龟背竹" |
| **浇水周期（天）** | AI 建议的浇水间隔天数 | 整数，如 3、7、14 |
| **分类** | AI 判断室内/室外 | indoor / outdoor |
| **备注** | 植物描述 + 养护建议（拼接） | 详见 4.3 |

### 4.2 AI 返回字段升级

| 字段 | 当前 | 升级后 | 说明 |
|------|------|--------|------|
| `name` | 植物中文名称 | **不变** → 自动填入「植物名称」 | 如"绿萝" |
| `watering_interval` | 浇水天数 | **不变** → 自动填入「浇水周期」 | 整数，如 7 |
| `category` | indoor/outdoor | **不变** → 自动填入「分类」 | 室内/室外 |
| `note` | 一句话建议 | **移除** | 被 description + care_tips 取代 |
| `description` | 无 | **新增** | 植物简介（品种、外观特征，2-3 句话） |
| `care_tips` | 无 | **新增** | 结构化养护建议，包含光照、温度、湿度、施肥、土壤、注意事项 |

### 4.3 备注字段自动填充格式

识别成功后，将 `description` 和 `care_tips` 拼接后写入「备注」：

```
【植物简介】
常绿藤本植物，叶片心形，翠绿有光泽，适应性强，是最常见的室内观叶植物之一。

【养护建议】
- 光照：喜散射光，避免强光直射
- 温度：适宜 15-28°C，不耐寒
- 湿度：喜湿润，可定期向叶面喷水
- 施肥：生长期每月施一次薄肥
- 土壤：疏松透气的腐殖土
- 注意：有轻微毒性，避免儿童和宠物误食
```

### 4.4 交互流程

```
用户点击 + → 弹窗 → 点击「📷 拍照识别植物」→ 拍照/选图
→ 图片压缩上传 → loading 状态 → AI 识别
→ 自动填入 4 个字段：
    ├── 植物名称 ← name
    ├── 浇水周期 ← watering_interval
    ├── 分类     ← category
    └── 备注     ← description + care_tips（拼接）
→ 用户可修改任意字段 → 保存
```

### 4.5 前端展示优化

- 备注字段 `<textarea>` 高度自适应，识别填充后自动撑高，确保内容可读
- 植物卡片的备注显示区支持多行，保留换行格式

### 4.6 PC 端适配设计

采用响应式布局，断点 `768px`，移动端保持现有样式不变，PC 端增强：

**整体布局：**
- 页面内容区最大宽度 `900px`，居中显示
- 两侧留白，背景色保持 `#f0f7f0`

**顶部标题栏：**
- 标题字号适当放大，最大宽度与内容区一致

**植物卡片列表：**
- 使用 CSS Grid 两列布局（`grid-template-columns: 1fr 1fr`）
- 卡片间距适当加大

**弹窗：**
- 弹窗从底部滑出改为 **居中显示**（`align-items: center`）
- 弹窗四角圆角（不再只有顶部圆角）
- 最大宽度 `500px`，保持当前值

**浮动按钮：**
- 位置不变，仍固定在右下角

## 5. 技术方案

### 5.1 后端修改（`app/routers/plants.py`）

**修改 `IdentifyResponse` 模型：**
```python
class IdentifyResponse(BaseModel):
    name: str
    watering_interval: int
    category: str
    description: str    # 新增：植物简介
    care_tips: str      # 新增：养护建议（已格式化的文本）
```

**修改 AI Prompt：**
- 要求返回 `description`（2-3 句植物简介）
- 要求返回 `care_tips`（结构化养护建议，包含光照/温度/湿度/施肥/土壤/注意事项）
- 移除原有的 `note` 字段

### 5.2 前端修改（`app/static/index.html`）

**自动填充逻辑修改：**
- 接收 `description` + `care_tips`，拼接为格式化文本填入备注
- 拼接格式：`【植物简介】\n{description}\n\n【养护建议】\n{care_tips}`
- 填充后自动调整 textarea 高度

**备注展示优化：**
- 植物卡片备注区保留换行（CSS `white-space: pre-line`）

**PC 端响应式适配：**
- 新增 `@media (min-width: 768px)` 媒体查询
- 内容区（提醒区、植物列表）最大宽度 `900px`，`margin: 0 auto` 居中
- 植物列表使用 CSS Grid 两列布局
- 弹窗改为垂直居中、四角圆角
- 标题栏字号适当放大

## 7. 验收标准

**AI 识别升级：**
1. 拍照识别后，**名称、浇水周期、分类** 正确填入对应表单字段
2. **备注** 字段自动填入「植物简介 + 养护建议」，格式清晰、包含光照/温度/湿度/施肥/土壤/注意事项
3. 备注 textarea 自动撑高，内容完整可见
4. 植物卡片列表中，备注内容保留换行格式，可读性好
5. 用户可手动修改所有字段后保存

**PC 端适配：**
6. PC 浏览器（宽度 >= 768px）打开时，内容区居中、最大宽度 900px，两侧有留白
7. 植物卡片列表在 PC 端呈两列排列
8. 弹窗在 PC 端垂直居中显示，四角圆角
9. 移动端样式不受影响，保持原有效果

**通用：**
10. 原有测试全部通过，不影响已有功能

## 8. 不包含的内容（Out of Scope）

- 不修改数据模型或数据库表结构
- 不新增依赖
