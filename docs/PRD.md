# Plant Sprite 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
Plant Sprite（植物精灵）

### 1.2 产品定位
一款以 AI 对话为核心的家庭生活助手微信小程序。用户可以通过自然语言与 AI 交流，咨询家庭场景中的各类问题（植物养护、家居清洁、收纳整理、宠物护理、烹饪技巧等），也可以使用工具箱中的实用工具（当前有植物管理）。AI 助手会根据用户的偏好设置进行个性化回复。

### 1.3 目标用户
- 家庭生活爱好者，需要日常生活建议的用户
- 家庭绿植、宠物养护新手
- 喜欢通过对话方式获取信息的用户

### 1.4 核心价值
- **AI 对话**：专注家庭生活场景，提供植物养护、家居清洁、烹饪等个性化建议，同时支持工具使用问答
- **工具箱**：提供植物管理（浇水提醒、拍照识别）等实用工具，后续可扩展更多家庭工具
- **个人中心**：管理个人信息和 AI 对话偏好设置

## 3. 功能详细设计

### 3.1 首页 — AI 对话（/pages/home/home）

#### 3.1.1 功能描述
对话式界面，用户通过文字输入与 AI 大模型交互。AI 专注于**家庭生活场景**的问题回答，包括但不限于：
- 🌿 植物养护：浇水、施肥、病虫害、光照建议等
- 🏠 家居清洁：清洁技巧、去污方法、收纳整理等
- 🐾 宠物护理：喂养、日常护理建议等
- 🍳 烹饪技巧：食材处理、菜谱建议、厨房小窍门等
- 🔧 工具使用：本小程序工具箱功能的使用方法和操作指引

AI 会根据用户在"我的"页面中设置的偏好（如家庭人数、是否有宠物、居住环境等）进行个性化回复。

#### 3.1.2 界面布局
```
┌─────────────────────────┐
│     Plant Sprite 🌱      │  ← 导航栏
├─────────────────────────┤
│                         │
│  🤖 你好！我是植物精灵    │  ← AI 欢迎语
│     有什么可以帮你的？    │
│                         │
│         👤 我的绿萝叶子    │  ← 用户消息（右对齐）
│            发黄怎么办？   │
│                         │
│  🤖 绿萝叶子发黄常见      │  ← AI 回复（左对齐）
│     原因有以下几种...     │
│                         │
│                         │
├─────────────────────────┤
│  [  输入消息...  ] [发送] │  ← 底部输入框
└─────────────────────────┘
```

#### 3.1.3 交互规则
- 消息列表自动滚动到最新消息
- AI 回复时显示"思考中..."的加载动画
- 支持上拉加载历史消息（本地存储最近 50 条）
- 输入框为空时发送按钮禁用
- 长按消息可复制文本

#### 3.1.4 后端接口
- `POST /api/chat`：发送用户消息，接收 AI 回复
  - 请求：`{ message: string }`
  - 响应：`{ reply: string }`
  - 使用通义千问大模型（已有 DASHSCOPE_API_KEY）
  - System prompt 中注入用户偏好设置，实现个性化回复
  - 限定回复范围：家庭生活场景 + 工具使用指引

## 4. 技术方案

### 4.1 前端改动

#### 4.1.1 新增页面
| 页面 | 路径 | 说明 |
|------|------|------|
| AI 对话 | pages/home/home | 新首页，对话界面 |
| 工具箱 | pages/tools/tools | 工具入口列表 |
| 个人中心 | pages/profile/profile | 用户信息与设置 |

#### 4.1.2 app.json 改动
- 新增 3 个 Tab 页路径
- 配置 `tabBar`（底部三栏导航）
- 登录页改为 `redirectTo` 跳转 Tab 页

#### 4.1.3 登录流程调整
- 当前：登录后 `redirectTo` → index 页
- 改为：登录后 `switchTab` → home 页（Tab 页不支持 redirectTo）

### 4.2 后端改动

#### 4.2.1 新增接口
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/chat | POST | AI 对话，调用通义千问文本模型 |
| /api/user/profile | GET | 获取用户信息 |
| /api/user/profile | PUT | 更新用户昵称/头像 |

#### 4.2.2 数据库改动
- User 表新增字段：`nickname`（String）、`avatar_url`（Text）、`preferences`（JSON/Text，存储 AI 对话偏好）
- preferences 字段结构：
  ```json
  {
    "chat_style": "gentle",
    "character": "cat",
    "custom_character": "",
    "nickname": "小主人"
  }
  ```

### 4.3 技术栈
- 前端：微信小程序原生框架
- 后端：FastAPI + SQLAlchemy（已有）
- AI 模型：通义千问（qwen-plus 文本 / qwen-vl-plus 图片识别，已有 DASHSCOPE_API_KEY）
- 部署：微信云托管（已有环境 prod-0g02is9d648082af）

## 6. 注意事项

1. **Tab 页限制**：微信小程序 Tab 页只能通过 `switchTab` 跳转，不能用 `navigateTo` 或 `redirectTo`
2. **登录页不在 Tab 中**：登录页保持为独立页面，登录成功后 `switchTab` 到首页
3. **原植物管理页面**：从 Tab 页降为普通页面，通过工具箱 `navigateTo` 进入
4. **AI 对话安全**：需设置合理的 system prompt，限制回复范围在家庭生活场景和工具使用指引，注入用户偏好实现个性化
5. **Token 管理**：所有 Tab 页的 `onShow` 都需检查登录状态
