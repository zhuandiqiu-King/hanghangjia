# PRD：Plant Sprite 微信小程序升级

> 版本：v2.0 | 日期：2026-03-11

---

## 1. 背景与现状

Plant Sprite 目前是一个基于 FastAPI + H5 单页的植物浇水提醒应用，具备以下能力：
- 植物增删改查（名称、浇水周期、分类、备注）
- 拍照识别植物（通义千问 VL），自动填充表单
- 浇水打卡 + 浇水记录查看
- 今日浇水提醒（逾期 / 即将到期）
- PC 端响应式适配

**当前限制：**
- 纯 Web 应用，用户需通过浏览器访问，无法推送通知
- 无用户体系，所有人共享同一份数据
- 不支持微信生态能力（分享、订阅消息等）
- 移动端体验不如原生小程序流畅

## 2. 需求目标

将 Plant Sprite 升级为**微信小程序**，实现：

1. **微信授权登录**：用户通过微信授权快速登录，数据按用户隔离
2. **功能平移**：将现有 H5 全部功能迁移到小程序原生页面
3. **后端升级**：新增用户模块，API 支持多用户数据隔离
4. **保留 H5**：原有 Web 版本继续可用（可选，作为降级方案）

## 3. 用户故事

> **故事 1（微信登录）：** 作为用户，我打开小程序后，点击"微信登录"按钮，授权后自动完成注册/登录，下次再打开直接进入首页，无需重复登录。

> **故事 2（数据隔离）：** 作为用户，我添加的植物只有我能看到和管理，不会和其他用户的数据混在一起。

> **故事 3（功能使用）：** 作为用户，我能在小程序中完成所有操作：添加植物、拍照识别、浇水打卡、查看提醒和浇水记录，体验和 H5 版一致。

## 5. 功能设计

### 5.1 微信授权登录

**登录流程：**
```
小程序启动
  → 检查本地 token 是否存在且有效
  → 有效：直接进入首页
  → 无效：显示登录页
      → 用户点击「微信登录」按钮
      → 调用 wx.login() 获取临时 code
      → 调用 wx.getUserProfile() 获取昵称和头像（需用户确认授权）
      → 将 code + 昵称 + 头像 发送到后端 /api/auth/login
      → 后端用 code 调用微信 code2Session 接口，获取 openid
      → 根据 openid 查找或创建用户
      → 返回 JWT token
      → 小程序存储 token 到本地（wx.setStorageSync）
      → 跳转首页
```

**登录页 UI：**

## 6. 后端改动

### 6.1 新增用户模型

**User 表：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int, PK | 主键 |
| `openid` | str, unique | 微信 openid，唯一标识 |
| `nickname` | str | 微信昵称 |
| `avatar_url` | str, nullable | 微信头像 URL |
| `created_at` | datetime | 注册时间 |

**Plant 表新增字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | int, FK → User.id | 植物所属用户（外键） |

### 6.2 新增 API

| 方法 | 路径 | 说明 | 请求 | 响应 |
|------|------|------|------|------|
| POST | `/api/auth/login` | 微信登录 | `{ code, nickname, avatar_url }` | `{ token, user: { id, nickname, avatar_url } }` |
| GET | `/api/auth/me` | 获取当前用户信息 | Header: `Authorization: Bearer <token>` | `{ id, nickname, avatar_url }` |

**登录接口逻辑：**
```
1. 接收小程序传来的 code
2. 用 code + appid + secret 请求微信 code2Session 接口
   GET https://api.weixin.qq.com/sns/jscode2session
     ?appid=APPID
     &secret=SECRET
     &js_code=CODE
     &grant_type=authorization_code
3. 获取 openid（+ session_key，不暴露给前端）
4. 查找 User：存在则更新昵称头像，不存在则创建
5. 生成 JWT token（payload 包含 user_id，过期时间 30 天）
6. 返回 token + 用户信息
```

### 6.3 现有 API 改动

所有植物相关接口增加用户鉴权，数据按 `user_id` 隔离：

| 接口 | 改动 |
|------|------|
| `POST /api/plants` | 从 token 取 user_id，创建时绑定 |
| `GET /api/plants` | 只返回当前用户的植物 |
| `GET /api/plants/{id}` | 校验植物属于当前用户 |
| `PUT /api/plants/{id}` | 校验植物属于当前用户 |
| `DELETE /api/plants/{id}` | 校验植物属于当前用户 |
| `POST /api/plants/{id}/water` | 校验植物属于当前用户 |
| `GET /api/plants/{id}/records` | 校验植物属于当前用户 |
| `GET /api/reminders` | 只返回当前用户的提醒 |
| `POST /api/plants/identify` | 无需改动（不涉及用户数据） |

**鉴权方式：**
- 请求 Header 携带 `Authorization: Bearer <JWT token>`
- 后端 FastAPI 依赖注入解析 token，获取 `user_id`
- 未携带或无效 token 返回 `401 Unauthorized`

### 6.4 新增环境变量

| 变量名 | 说明 |
|--------|------|
| `WX_APPID` | 微信小程序 AppID |

## 8. 技术要点

### 8.1 图片处理（拍照识别）

```
wx.chooseMedia()
  → 获取临时文件路径 tempFilePath
  → wx.getFileSystemManager().readFile() 读取为 base64
  → 压缩处理（canvas 绘制缩放）
  → wx.request() POST 到 /api/plants/identify
  → 自动填充表单
```

### 8.2 网络请求封装

`utils/api.js` 统一封装：
- 自动添加 `Authorization` Header
- 401 响应自动跳转登录页
- 统一错误提示（`wx.showToast`）
- 基础 URL 配置（指向公网后端）

### 8.3 登录态管理

`utils/auth.js`：
- `getToken()` / `setToken()` / `clearToken()`：基于 `wx.getStorageSync`
- `checkLogin()`：检查 token 是否存在，用于页面 `onLoad` 中判断
- token 过期后自动跳转登录页重新授权

### 8.4 数据库迁移

Plant 表新增 `user_id` 列：
- 新列设为 nullable，保留旧数据兼容性
- 旧数据 `user_id = NULL` 不影响新用户使用
