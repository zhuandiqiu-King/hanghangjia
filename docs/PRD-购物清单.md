# 购物清单 产品需求文档（PRD）

## 1. 功能概述

### 1.1 功能定位
家庭共享购物清单工具，帮助家庭成员协作管理日常采购需求。支持快速添加、分类管理、多人协作、智能建议等能力。

### 1.2 目标用户
- 需要管理家庭日常采购的用户
- 家庭成员之间需要共享购物需求的场景
- 习惯提前规划采购内容的用户

### 1.3 核心价值
- **快速记录**：随时随地添加购物项，支持语音输入和 AI 智能拆分
- **家庭共享**：同一家庭的成员共享清单，实时同步
- **分类清晰**：按商品类别自动分组，购物时一目了然
- **历史复用**：常买商品一键添加，减少重复操作

---

## 2. 信息架构

### 2.1 页面层级

```
工具箱 → 购买清单
├── 清单主页 (shopping/index)     ← 当前清单 + 快速添加
├── 历史记录 (shopping/history)   ← 已完成的清单归档
└── 常买清单 (shopping/frequent)  ← 高频商品快速添加
```

### 2.2 入口
- 工具箱 → 购买清单（主入口）
- 首页快捷入口 → 购物清单
- AI 聊天中对话添加（后续迭代）

## 3. 功能详细设计

### 3.1 清单主页（shopping/index）

#### 3.1.1 页面结构

```
┌──────────────────────────────┐
│  🛒 购物清单          [历史]  │
├──────────────────────────────┤
│  ┌────────────────────────┐  │
│  │ 🎤 添加购物项...    [+] │  │
│  └────────────────────────┘  │
│                              │
│  🥬 生鲜果蔬 (3)             │
│  ┌────────────────────────┐  │
│  │ ○ 西红柿     2斤   ¥8  │  │
│  │ ○ 鸡蛋       1盒  ¥15  │  │
│  │ ○ 牛奶       2瓶       │  │
│  └────────────────────────┘  │
│                              │
│  🧴 日用百货 (1)             │
│  ┌────────────────────────┐  │
│  │ ○ 洗洁精     1瓶       │  │
│  └────────────────────────┘  │
│                              │
│  ✅ 已买 (2)        [清空]   │
│  ┌────────────────────────┐  │
│  │ ✓ 面包              小明 │  │
│  │ ✓ 矿泉水            小红 │  │
│  └────────────────────────┘  │
│                              │
│  [🗑 清空已买]  [📋 一键复制] │
└──────────────────────────────┘
```

#### 3.1.2 功能说明

| 功能 | 说明 |
|------|------|
| **快速添加** | 输入框输入商品名，回车即添加。支持一次输入多个，用逗号分隔（如"牛奶,鸡蛋,面包"） |
| **语音添加** | 长按麦克风按钮，语音识别后自动拆分为多个商品项 |
| **AI 智能拆分** | 输入"明天做红烧肉需要的食材"，AI 自动拆解为猪五花、酱油、冰糖、姜等 |
| **分类展示** | 商品按类别分组显示（生鲜果蔬/肉禽蛋奶/日用百货/零食饮料/其他） |
| **勾选完成** | 点击商品前的圆圈标记为"已买"，移到底部已买区域 |
| **操作人记录** | 勾选时自动记录是哪个家庭成员买的 |
| **数量和价格** | 可选填数量和预估价格，方便预算控制 |
| **左滑操作** | 左滑商品项，显示"编辑"和"删除"按钮 |
| **清空已买** | 一键清空已完成的商品，归入历史记录 |
| **一键复制** | 将当前清单复制为文本，方便分享到微信聊天 |

#### 3.1.3 商品分类规则

| 类别 | 图标 | 关键词示例 |
|------|------|-----------|
| 生鲜果蔬 | 🥬 | 西红柿、苹果、白菜、土豆、香蕉 |
| 肉禽蛋奶 | 🥩 | 猪肉、鸡蛋、牛奶、虾、鱼 |
| 粮油调味 | 🍚 | 大米、面粉、酱油、盐、油 |
| 零食饮料 | 🍪 | 饼干、薯片、可乐、果汁、酸奶 |
| 日用百货 | 🧴 | 洗洁精、纸巾、牙膏、垃圾袋 |
| 其他 | 📦 | 未匹配到以上类别的商品 |

> 分类逻辑：前端维护关键词表做初步分类，AI 智能添加时由后端直接返回分类。

### 3.2 添加商品弹窗

#### 3.2.1 页面结构

```
┌──────────────────────────┐
│  添加商品                  │
│                           │
│  名称*   [西红柿        ]  │
│  数量    [2     ] [斤 ▾]  │
│  预估价  [¥ 8          ]  │
│  分类    [🥬 生鲜果蔬  ▾]  │
│  备注    [要红的熟一点   ]  │
│                           │
│  [取消]        [确认添加]  │
└──────────────────────────┘
```

#### 3.2.2 字段说明

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| 名称 | 是 | 文本 | 商品名称，最长 50 字 |
| 数量 | 否 | 数字+单位 | 如 2斤、1盒、3瓶。单位可选：个/斤/克/瓶/盒/袋/包 |
| 预估价 | 否 | 数字 | 预估价格，用于预算统计 |
| 分类 | 否 | 下拉选择 | 自动识别，用户可手动修改 |
| 备注 | 否 | 文本 | 补充说明，如品牌偏好、规格要求 |

### 3.3 历史记录（shopping/history）

#### 3.3.1 功能说明

| 功能 | 说明 |
|------|------|
| **按日期分组** | 展示每次"清空已买"后归档的购物记录，按日期倒序 |
| **查看详情** | 显示商品列表、购买人、总花费 |
| **一键重买** | 将某次历史记录中的所有商品重新加入当前清单 |
| **花费统计** | 显示本周/本月的购物总花费（如填写了价格） |

### 3.4 常买清单（shopping/frequent）

#### 3.4.1 功能说明

| 功能 | 说明 |
|------|------|
| **自动统计** | 系统根据购买频次自动排序常买商品 |
| **快速添加** | 点击常买商品一键加入当前清单 |
| **批量添加** | 勾选多个常买商品，一次性加入清单 |
| **管理** | 长按删除不需要的常买商品 |

## 4. 数据模型

### 4.1 数据库表设计

```
ShoppingList（购物清单）
├── id              主键
├── family_id       所属家庭
├── name            清单名称（默认"购物清单"）
├── status          状态：active / archived
├── created_at      创建时间
└── archived_at     归档时间

ShoppingItem（购物项）
├── id              主键
├── list_id         所属清单
├── name            商品名称
├── quantity         数量（如 "2斤"）
├── price           预估价格
├── category        分类（fresh/meat/grain/snack/daily/other）
├── note            备注
├── is_bought       是否已买
├── bought_by       购买人（user_id）
├── bought_at       购买时间
├── created_by      添加人（user_id）
└── created_at      创建时间

FrequentItem（常买商品）
├── id              主键
├── family_id       所属家庭
├── name            商品名称
├── category        分类
├── buy_count       购买次数
└── last_bought_at  最近购买时间
```

### 4.2 关键关系
- 一个家庭同时只有一个 active 清单，完成后归档
- ShoppingItem 归属 ShoppingList
- FrequentItem 由系统自动统计生成

## 5. API 设计

### 5.1 清单管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/shopping/current | 获取当前活跃清单（含商品列表） |
| POST | /api/shopping/items | 添加商品（支持批量） |
| PUT | /api/shopping/items/{id} | 编辑商品 |
| DELETE | /api/shopping/items/{id} | 删除商品 |
| PUT | /api/shopping/items/{id}/buy | 标记已买 |
| PUT | /api/shopping/items/{id}/unbuy | 取消已买 |
| POST | /api/shopping/archive | 归档当前清单（清空已买） |

### 5.2 历史与常买

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/shopping/history | 历史清单列表 |
| GET | /api/shopping/history/{id} | 历史清单详情 |
| POST | /api/shopping/history/{id}/rebuy | 一键重买 |
| GET | /api/shopping/frequent | 常买商品列表 |
| POST | /api/shopping/frequent/add-to-list | 常买商品加入当前清单 |

### 5.3 智能添加

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/shopping/smart-add | AI 智能拆分（输入一句话，返回商品列表） |

## 6. 交互细节

### 6.1 快速添加流程

```
用户在输入框输入 "牛奶, 鸡蛋, 面包"
  → 前端按逗号拆分
  → 批量调 POST /api/shopping/items
  → 自动分类，加入清单
  → 输入框清空，清单刷新
```

### 6.2 AI 智能添加流程

```
用户输入 "明天做红烧肉需要什么"
  → 调 POST /api/shopping/smart-add
  → AI 返回: [{name:"猪五花", quantity:"1斤", category:"meat"}, ...]
  → 展示确认弹窗，用户可勾选/修改
  → 确认后批量加入清单
```

### 6.3 勾选已买流程

```
用户点击商品前的 ○
  → 调 PUT /api/shopping/items/{id}/buy
  → 记录 bought_by = 当前用户
  → 商品移到 "已买" 区域，显示购买人昵称
  → 更新 FrequentItem 的 buy_count
```

### 6.4 归档流程

```
用户点击 "清空已买"
  → 确认弹窗
  → 已买商品归入历史记录
  → 未买商品保留在当前清单
  → 如果清单全部清空，自动创建新的空清单
```
